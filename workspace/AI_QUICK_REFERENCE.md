## 一、核心操作

### 1. 设置面板
**入口**: 右上角 ⚙️ 图标

**主要配置项**:
- **LLM 提供商**: 选择模型 → 填写 API Key → 保存
- **渠道**: 选择渠道（飞书/QQ/微信等）→ 填写配置 → 测试连接 → 保存
- **模型参数**: 调整温度(0-2)、Max Tokens 等 → 保存
- **人设**: 选择预设或自定义系统提示词 → 保存
- **工作空间**: 选择文件夹 → 保存（所有文件操作的默认目录）
- **配置管理**: 导出/导入配置备份

**支持的 LLM 提供商**:
| 提供商 | 需要配置 | 获取地址 |
|--------|----------|----------|
| OpenAI | API Key | https://platform.openai.com |
| Claude (Anthropic) | API Key | https://console.anthropic.com |
| 智谱 AI (GLM) | API Key | https://open.bigmodel.cn |
| 阿里云 (通义千问) | API Key | https://dashscope.aliyun.com |
| 百度云 (文心一言) | API Key, Secret Key | https://console.bce.baidu.com/qianfan |
| 字节跳动 (豆包) | API Key | https://console.volcengine.com |
| 月之暗面 (Kimi) | API Key | https://platform.moonshot.cn |
| MiniMax | API Key, Group ID | https://platform.minimaxi.com |
| DeepSeek | API Key | https://platform.deepseek.com |
| Coding Plan 套餐 | 套餐配置 | 内置预设 |

**配置步骤**:
1. 点击右上角 ⚙️ → LLM 提供商
2. 从下拉列表选择提供商
3. 填写对应的 API Key（部分需要额外参数）
4. 点击"测试连接"验证配置
5. 测试成功后点击"保存"

**渠道必填字段速查**:
| 渠道 | 必填字段 |
|------|----------|
| 飞书 | App ID, App Secret, Verification Token, Encrypt Key |
| QQ频道 | App ID, Token, Secret |
| 微信 | App ID, App Secret, Token, EncodingAESKey |

---

### 2. 技能管理

**入口**: 右上角 ⚙️ 设置图标 → 左侧菜单"技能"

**基本操作**:
- **启用/禁用**: 点击技能卡片开关
- **配置**: 点击卡片 → 编辑 config.json → 保存
- **查看文档**: 点击卡片 → 查看文档（SKILL.md）
- **新建**: 新建技能按钮 或 手动在 `skills/` 目录创建
- **删除**: 点击删除图标 → 确认

**重要概念**: 技能是包含命令行示例的文档，使用时需先读取文档再执行命令。

---

### 3. 定时任务

**入口**: 右上角 ⚙️ 设置图标 → 左侧菜单"定时任务"

**基本操作**:
- **创建**: 新建任务 → 填写名称、Cron表达式、提示词 → 选择技能 → 创建
- **启用/禁用**: 点击任务开关
- **编辑**: 点击任务卡片 → 修改 → 保存
- **手动执行**: 点击"立即执行"
- **查看历史**: 点击任务 → 执行历史

**Cron 表达式速查** (`秒 分 时 日 月 周`):
- `0 0 9 * * *` - 每天 9:00
- `0 */30 * * * *` - 每 30 分钟
- `0 0 */2 * * *` - 每 2 小时
- `0 0 9 * * 1` - 每周一 9:00

---

### 4. 多智能体团队

**入口**: 右上角 ⚙️ 设置图标 → 左侧菜单"多智能体"

**基本操作**:
- **创建**: 新建团队 → 填写名称、描述 → 选择工作模式 → 添加成员 → 创建
- **使用**: 对话窗口顶部选择团队 → 输入任务
- **编辑**: 点击团队卡片 → 修改 → 保存
- **启用/禁用**: 点击团队开关

**工作模式**:
- **流水线**: 成员按顺序处理（适合明确流程）
- **依赖图**: 根据依赖关系并行/串行（适合复杂工作流）
- **多视角**: 成员独立分析后汇总（适合多角度评估）

## 三、故障排查

### 日志系统

**日志位置**: `data/logs/` 和 `data/audit_logs/`

**日志类型**:
1. **应用日志**: `data/logs/CountBot_YYYY-MM-DD.log`
   - 包含应用运行的所有信息
   - 每天自动归档，旧日志会压缩为 .zip
   
2. **错误日志**: `data/logs/error_YYYY-MM-DD.log`
   - 仅包含错误和异常信息
   - 用于快速定位问题

3. **渠道日志**: `data/logs/feishu_worker_*.log`
   - 各渠道（飞书/QQ/微信等）的独立日志
   - 包含消息收发记录

4. **审计日志**: `data/audit_logs/audit_YYYY-MM-DD_*.log`
   - 记录所有工具调用和 AI 响应
   - JSON 格式，每行一条记录
   - **需要在设置中开启"安全审计"功能**

**审计日志格式**:
```json
{
  "id": "工具调用ID",
  "timestamp": "时间戳",
  "tool_name": "工具名称",
  "arguments": {"参数": "值"},
  "session_id": "会话ID",
  "status": "pending/success/error",
  "result": "执行结果",
  "error": "错误信息",
  "duration_ms": 执行时长
}
```

**查看工具执行历史**:
- API 接口: `GET /api/tools/conversation-history`
- 参数:
  - `session_id`: 按会话过滤
  - `tool_name`: 按工具名称过滤
  - `limit`: 返回数量（默认50）
  - `offset`: 分页偏移
- 注意: 返回结果会自动过滤隐藏工具（read_file, write_file等）

**开启审计日志**:
1. 点击右上角 ⚙️ 设置图标
2. 找到"安全审计"选项
3. 开启审计功能
4. 审计日志会自动保存到 `data/audit_logs/`

### 1. 技能不工作
**诊断流程**:
1. 检查技能是否启用（设置 → 技能 → 开关为绿色）
2. 检查配置是否正确（点击技能卡片查看config.json）
3. 安装依赖: `pip install -r skills/{技能名}/requirements.txt`
4. 查看错误日志定位问题

**常见错误**:
- "技能未找到" → 技能未启用或文件夹不存在
- "配置错误" → config.json 格式错误或缺少必填字段
- "命令执行失败" → Python 环境问题或依赖未安装

---

### 2. 定时任务不执行
**诊断流程**:
1. 确认任务开关为绿色（设置 → 定时任务）
2. 验证Cron表达式（使用 https://crontab.guru/）
3. 检查调度器: `python backend/scripts/check_cron_jobs.py`
4. 查看执行历史（点击任务 → 执行历史）
5. 确认相关技能已启用

**常见原因**:
- Cron 表达式错误（最常见）
- 任务开关未打开
- 后台调度器未启动（重启应用解决）
- 技能未启用或配置错误

---

### 3. 渠道消息收不到
**诊断流程**:
1. 确认渠道配置已保存（设置 → 渠道）
2. 点击"测试连接"验证配置
3. 检查机器人是否在群组中且有权限
4. 查看日志: `botpy.log`
5. 重启渠道服务或应用

---

### 4. LLM 不工作
**完整诊断流程**:

**步骤 1: 检查配置是否保存**
- 路径: 设置 → LLM 提供商
- 确认 API Key 已填写
- 点击"保存"按钮（重要！）
- 刷新页面确认配置仍然存在

**步骤 2: 验证 API Key 格式**
- OpenAI: `sk-...` 开头
- Claude: `sk-ant-...` 开头
- 智谱 AI: 长字符串
- 阿里云: `sk-...` 开头
- 百度云: 需要 API Key 和 Secret Key
- Kimi: `sk-...` 开头
- 检查是否有多余空格或换行

**步骤 3: 测试连接**
- 点击"测试连接"按钮
- 查看返回的错误信息
- 常见错误:
  - "Invalid API Key" → API Key 错误或过期
  - "Network Error" → 网络问题或需要代理
  - "Rate Limit" → 超出配额限制
  - "Model Not Found" → 模型名称错误

**步骤 4: 检查模型配置**
- 路径: 设置 → 模型配置
- 确认模型名称正确
- 不同提供商的模型名称:
  - OpenAI: `gpt-4`, `gpt-3.5-turbo`
  - Claude: `claude-3-opus`, `claude-3-sonnet`
  - 智谱: `glm-4`, `glm-3-turbo`
  - 阿里云: `qwen-max`, `qwen-turbo`
  - 百度: `ernie-bot-4`, `ernie-bot-turbo`
  - Kimi: `moonshot-v1-8k`, `moonshot-v1-32k`

**步骤 5: 检查网络连接**
- 国外服务（OpenAI、Claude）可能需要代理
- 测试网络: `ping api.openai.com`
- 检查防火墙设置

**步骤 6: 查看错误日志**
- 应用日志: `data/logs/CountBot_YYYY-MM-DD.log`
- 错误日志: `data/logs/error_YYYY-MM-DD.log`
- 搜索关键词: "API", "LLM", "provider"

**常见问题**:
- **配置未保存**: 填写后忘记点击"保存"
- **API Key 输入错误**: 复制时包含空格或换行
- **模型名称错误**: 使用了不存在的模型
- **网络问题**: 无法访问 API 端点
- **配额用尽**: API 调用次数或余额不足
- **权限问题**: API Key 没有对应模型的权限

---

### 5. AI 响应慢
**优化方法**:
- 降低 Max Tokens（设置 → 模型配置）
- 切换到更快的模型（如 GPT-3.5）
- 清理旧记忆文件（`data/memory/`）
- 调整温度参数（精确任务用 0-0.3）

---

### 6. 数据库问题
**修复步骤**:
1. 备份: 复制 `data/countbot.db`
2. 运行修复: `python fix_database_now.py`
3. 或删除数据库文件，重启应用自动创建新库

---

### 7. 记忆系统问题
**常见问题**:
- **记忆文件在哪**: `data/memory/` 目录，每个会话一个文件
- **记忆格式**: MEMORY.md 纯文本文件
- **清理记忆**: 删除对应的记忆文件或要求 AI "忘记某事"
- **记忆不生效**: 检查记忆文件是否存在，AI 会自动判断是否需要记忆

---

### 8. 工作空间问题
**常见问题**:
- **工作空间是什么**: AI 执行文件操作和命令的默认目录
- **如何设置**: 设置 → 工作空间 → 选择文件夹
- **默认位置**: 应用安装目录
- **临时文件**: 存储在 `{工作空间}/temp/` 目录

---

### 9. 会话管理问题
**常见操作**:
- **新建会话**: 左侧会话列表 → + 按钮
- **切换会话**: 点击会话列表中的会话
- **重命名会话**: 右键会话 → 重命名
- **删除会话**: 右键会话 → 删除（会同时删除记忆）
- **搜索消息**: 对话窗口顶部 → 🔍 搜索图标

---

### 10. 性能优化
**优化建议**:
- **响应慢**: 降低 Max Tokens、切换更快模型、清理记忆文件
- **内存占用高**: 清理 `data/temp/` 临时文件
- **日志文件大**: 定期清理 `data/logs/` 旧日志（自动压缩）
- **数据库膨胀**: 定期备份并重建数据库

**注意事项**:

- 读取时注意控制返回内容长度，避免输出过多
- 可以使用 `tail` 命令只读取最后几行
- API 返回结果已过滤隐藏工具（read_file, write_file等）

### 智能提示

**AI 应该知道的**:

1. **工具选择**:
   - 文件操作优先用 `read_file`、`write_file`、`edit_file`
   - 命令执行用 `exec`
   - 网页内容用 `web_fetch`
   - 长时间任务用 `spawn` 创建子代理

2. **错误处理**:
   - 工具调用失败时，查看错误信息并尝试其他方法
   - 文件不存在时，先用 `list_dir` 确认路径
   - 命令执行失败时，检查工作空间和权限

3. **性能优化**:
   - 大文件分段读取（使用 start_line 和 end_line）
   - 避免重复调用相同工具
   - 合理使用记忆系统，减少重复询问

4. **用户体验**:
   - 工具调用前简要说明要做什么
   - 工具调用后总结结果
   - 遇到问题时主动询问用户

### 快速诊断决策树

**用户报告问题时，按此流程诊断**:

```
问题类型判断
├─ 功能不工作
│  ├─ 技能不工作 → 检查启用状态 → 检查配置 → 查看日志
│  ├─ 定时任务不执行 → 检查开关 → 验证Cron → 检查调度器
│  ├─ 渠道消息收不到 → 检查配置 → 测试连接 → 查看渠道日志
│  ├─ LLM不工作 → 检查配置保存 → 验证API Key → 测试连接 → 检查模型名称
│  └─ 工具调用失败 → 查看审计日志 → 检查权限 → 检查工作空间
│
├─ 性能问题
│  ├─ 响应慢 → 降低Max Tokens → 切换模型 → 清理记忆
│  ├─ 内存占用高 → 清理临时文件 → 清理日志 → 重启应用
│  └─ 数据库慢 → 备份重建 → 清理旧数据
│
├─ 配置问题
│  ├─ LLM配置失败 → 检查API Key格式 → 检查是否保存 → 测试连接 → 查看错误日志
│  ├─ 渠道配置失败 → 检查必填字段 → 测试连接 → 查看错误日志
│  └─ 工作空间问题 → 检查路径 → 检查权限 → 重新设置
│
└─ 数据问题
   ├─ 记忆丢失 → 检查记忆文件 → 检查会话ID
   ├─ 会话丢失 → 检查数据库 → 备份恢复
   └─ 数据库损坏 → 运行修复脚本 → 备份恢复
```

### AI 智能判断指南

**当用户描述问题时，AI 应该**:

1. **识别问题类型**:
   - 功能位置询问 → 查阅本文档第一部分
   - 功能不工作 → 查阅故障排查章节
   - 性能问题 → 查阅性能优化建议
   - 配置问题 → 提供具体配置步骤

2. **主动收集信息**:
   - 询问具体错误信息
   - 询问最近的操作
   - 询问是否查看过日志
   - 询问配置是否正确

3. **提供解决方案**:
   - 先给最可能的原因
   - 提供具体操作步骤
   - 提供备选方案
   - 询问是否需要查看日志

4. **使用工具辅助**:
   - 可以直接读取日志文件
   - 可以调用 API 查看工具历史
   - 可以检查配置文件
   - 可以执行诊断命令

**示例对话**:

用户: "定时任务不执行"
AI: 
```
我来帮你诊断定时任务问题。请先确认：

1. 任务开关是否为绿色（已启用）？
   - 路径：设置 → 定时任务 → 找到任务

2. Cron 表达式是否正确？
   - 可以用 https://crontab.guru/ 验证

3. 让我检查一下调度器状态...
   [调用 exec 执行诊断命令]

根据检查结果，问题可能是...
```
