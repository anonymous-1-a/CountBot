# CountBot v0.5.0 发布说明

**发布日期**: 2026年3月19日  
**版本号**: 0.5.0

---

## 核心亮点

### 1. 真正的智能体团队正式成型

`v0.5.0` 的重点不再是单个 Agent 的功能叠加，而是把 CountBot 从“单助手系统”升级为“可编排的智能体团队”：

- **多角色协作** - 不同角色可以承担规划、执行、审阅、总结等职责
- **团队级编排** - 支持按工作流组织多个 Agent 的顺序、依赖和条件流转
- **上下文连续传递** - 团队成员之间共享必要上下文，减少重复沟通
- **更稳定的任务拆解** - 复杂任务可以分解给不同角色完成，提升可控性

这意味着 CountBot 已经可以从“一个 AI 聊天助手”，升级为“一个能分工协作的智能体团队运行环境”。

### 2. 配置系统升级为角色级、团队级、多机器人级

在 `v0.4.0` 的会话级配置基础上，`v0.5.0` 进一步强调多角色和多机器人场景：

- **角色级配置** - 为不同角色单独指定模型、提示词和行为风格
- **团队级配置** - 团队内不同 Agent 可以使用不同提供商和不同模型
- **会话级隔离** - 每个会话仍然可保持独立配置与运行状态
- **多机器人配置** - 不同渠道和不同入口可以绑定不同机器人配置

一个工作区中，你可以同时维护：

- 编程团队
- 文案团队
- 客服机器人
- 渠道专用机器人
- 私有助理角色组合

### 3. 自定义渠道能力显著增强

`v0.5.0` 持续强化渠道层，使 CountBot 更适合作为统一接入层运行：

- **自定义渠道扩展更清晰** - 渠道管理、消息处理、媒体发送链路更完整
- **多个机器人接入能力增强** - 支持不同渠道使用不同机器人配置
- **渠道能力继续扩展** - 微博、企业微信、小智 AI 等场景能力持续完善
- **会话与渠道绑定更自然** - 降低跨渠道运行时的上下文混乱

这让 CountBot 不只是支持“多个 IM 渠道”，而是开始具备“按渠道组织多个机器人”的能力。

### 4. 前端全面重构与体验升级

这一版本的前端不是简单修补，而是一次比较完整的体验升级：

- **聊天界面重构** - 消息展示、工具调用、推理块、会话切换体验更清晰
- **团队与配置面板增强** - 团队、角色、渠道、提供商、技能配置更直观
- **工作流展示更完整** - 更适合展示多角色、多步骤任务过程
- **国际化与主题优化** - 中英文文案、暗色主题和细节样式进一步统一
- **交互效率提升** - 复杂配置场景下的可读性和操作效率更高

目标很明确：让复杂系统不只“能配”，而是“配得明白、看得清楚、用得顺手”。

### 5. 后端运行链路继续增强

围绕多智能体、多渠道、多配置这三条主线，后端继续做了深层优化：

- **Agent Loop / Context / Subagent / Workflow 持续增强**
- **渠道 Handler / Manager / Send Media 链路增强**
- **会话运行时配置与配置校验继续完善**
- **工具调用上下文、消息流和任务执行过程更稳定**

这部分升级的价值在于，复杂场景不再只是“理论支持”，而是更接近长期稳定运行。

---

## 这次更新适合谁

- 想把 CountBot 从单助手升级为多角色团队的人
- 需要在不同渠道接入多个机器人的团队
- 希望为不同角色配置不同模型、不同提示词、不同职责的人
- 需要更清晰前端配置界面和更完整工作流展示的人

---

## 典型场景

### 编程团队

- `Planner` 负责任务拆解
- `Coder` 负责实现
- `Reviewer` 负责检查风险和回归问题

### 渠道机器人矩阵

- 企业微信机器人负责内部通知
- QQ / 微博机器人负责外部互动
- Web 端保留完整配置和调试能力

### 多角色内容生产

- 选题角色负责策划
- 写作角色负责初稿
- 审校角色负责风格统一和事实检查

---

## 升级摘要

从产品能力上看，`v0.5.0` 的核心变化可以概括为：

- 从单 Agent 能力，升级到真正的智能体团队能力
- 从单一会话配置，升级到多角色、多团队、多机器人配置
- 从“支持多个渠道”，升级到“可按渠道组织多个机器人”
- 从“功能可用”，升级到“前后端整体体验显著优化”
