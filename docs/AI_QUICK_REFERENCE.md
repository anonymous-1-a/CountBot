# AI 快速参考手册

> 本文档为 AI 助手提供快速、准确的用户指引。当用户询问功能位置、使用方法或遇到问题时使用。

## 第一部分：UI 操作完整指南

### 1. 设置功能操作路径

#### 1.1 打开设置面板
**操作**: 点击界面右上角的 ⚙️ 设置图标
**结果**: 打开设置侧边栏，显示所有配置选项

#### 1.2 配置 LLM 提供商
**完整路径**: 
1. 点击右上角 ⚙️ 设置图标
2. 在左侧菜单选择"LLM 提供商"
3. 从下拉列表选择提供商（OpenAI、Claude、Gemini、本地模型等）
4. 填写对应的 API Key 或端点地址
5. 点击"保存"按钮

**常见提供商配置**:
- OpenAI: 需要 API Key
- Claude: 需要 API Key  
- Azure OpenAI: 需要 Endpoint、API Key、Deployment Name
- 本地模型: 需要 Base URL（如 http://localhost:11434）

#### 1.3 配置渠道（飞书/QQ/微信等）
**完整路径**:
1. 点击右上角 ⚙️ 设置图标
2. 在左侧菜单选择"渠道"
3. 选择要配置的渠道标签页（飞书、QQ频道、微信公众号、钉钉、Slack、Discord）
4. 填写该渠道所需的配置信息
5. 点击"测试连接"按钮验证配置
6. 测试成功后点击"保存"按钮

**各渠道必填字段**:
- 飞书: App ID、App Secret、Verification Token、Encrypt Key
- QQ 频道: App ID、Token、Secret
- 微信公众号: App ID、App Secret、Token、EncodingAESKey
- 钉钉: App Key、App Secret、Agent ID
- Slack: Bot Token、Signing Secret
- Discord: Bot Token


#### 1.4 配置模型参数
**完整路径**:
1. 点击右上角 ⚙️ 设置图标
2. 在左侧菜单选择"模型配置"
3. 调整以下参数：
   - 温度（Temperature）: 0.0-2.0，控制创造性
   - 最大 Token（Max Tokens）: 控制回复长度
   - Top P: 控制采样范围
   - 频率惩罚（Frequency Penalty）: 减少重复
   - 存在惩罚（Presence Penalty）: 鼓励新话题
4. 点击"保存"按钮

**参数建议**:
- 代码生成: 温度 0.3-0.5
- 创意写作: 温度 0.7-1.0
- 精确任务: 温度 0.0-0.3

#### 1.5 配置人设/性格
**完整路径**:
1. 点击右上角 ⚙️ 设置图标
2. 在左侧菜单选择"人设"
3. 选择预设性格或选择"自定义"
4. 如果选择自定义，在文本框输入系统提示词
5. 点击"保存"按钮

**预设性格**: 专业助手、友好伙伴、严谨专家、创意顾问、技术极客

#### 1.6 配置工作空间
**完整路径**:
1. 点击右上角 ⚙️ 设置图标
2. 在左侧菜单选择"工作空间"
3. 点击"选择文件夹"按钮
4. 在文件选择器中选择工作目录
5. 点击"保存"按钮

**说明**: 配置后，所有文件操作和技能执行都会在此目录下进行

#### 1.7 导出/导入配置
**导出配置**:
1. 点击右上角 ⚙️ 设置图标
2. 在左侧菜单选择"配置管理"
3. 点击"导出配置"按钮
4. 选择保存位置，文件名如 `config_backup_20260303.json`

**导入配置**:
1. 点击右上角 ⚙️ 设置图标
2. 在左侧菜单选择"配置管理"
3. 点击"导入配置"按钮
4. 选择之前导出的 JSON 文件
5. 系统会自动应用配置并重启相关服务

### 2. 技能管理操作路径

#### 2.1 查看技能列表
**操作**: 点击界面左侧的 🔧 技能图标
**结果**: 打开技能面板，显示所有已安装的技能

#### 2.2 启用/禁用技能
**完整路径**:
1. 点击左侧 🔧 技能图标
2. 在技能列表中找到目标技能
3. 点击技能卡片右上角的开关按钮
4. 开关变绿表示已启用，变灰表示已禁用

**说明**: 禁用的技能不会出现在 AI 的可用工具列表中

#### 2.3 配置技能参数
**完整路径**:
1. 点击左侧 🔧 技能图标
2. 点击要配置的技能卡片
3. 在弹出的配置编辑器中修改 JSON 配置
4. 点击"保存"按钮
5. 系统会自动验证配置格式

**配置文件位置**: `skills/{技能名}/config.json`

**常见配置项**:
- API Key: 第三方服务的密钥
- Base URL: API 端点地址
- Timeout: 超时时间（秒）
- Headless: 浏览器是否无头模式（true/false）

#### 2.4 新建技能
**方法一：通过 UI 创建**
1. 点击左侧 🔧 技能图标
2. 点击"新建技能"按钮
3. 填写技能名称、描述
4. 选择技能模板（可选）
5. 点击"创建"按钮

**方法二：手动添加**
1. 在 `skills/` 目录下创建新文件夹，如 `skills/my-skill/`
2. 创建必需文件：
   - `SKILL.md`: 技能说明文档
   - `config.json`: 配置文件
   - `scripts/`: 脚本目录
3. 重启应用或刷新技能列表

#### 2.5 查看技能文档
**完整路径**:
1. 点击左侧 🔧 技能图标
2. 点击技能卡片
3. 点击"查看文档"按钮
4. 在新窗口中查看 SKILL.md 内容

**说明**: 技能文档包含详细的使用方法、命令示例、配置说明

#### 2.6 删除技能
**完整路径**:
1. 点击左侧 🔧 技能图标
2. 找到要删除的技能
3. 点击技能卡片上的删除图标（垃圾桶）
4. 确认删除操作

**说明**: 删除会移除技能文件夹，无法恢复

### 3. 定时任务操作路径

#### 3.1 查看定时任务列表
**操作**: 点击界面左侧的 ⏰ 定时任务图标
**结果**: 打开定时任务面板，显示所有任务

#### 3.2 创建新的定时任务
**完整路径**:
1. 点击左侧 ⏰ 定时任务图标
2. 点击"新建任务"按钮
3. 填写任务信息：
   - 任务名称: 描述性名称
   - Cron 表达式: 如 `0 0 9 * * *`（每天 9:00）
   - 提示词: AI 要执行的任务描述
   - 选择技能: 任务需要用到的技能（可选）
4. 点击"创建"按钮

**Cron 表达式格式**: `秒 分 时 日 月 周`
- `0 0 9 * * *`: 每天 9:00
- `0 */30 * * * *`: 每 30 分钟
- `0 0 */2 * * *`: 每 2 小时
- `0 0 9 * * 1`: 每周一 9:00

#### 3.3 启用/禁用定时任务
**完整路径**:
1. 点击左侧 ⏰ 定时任务图标
2. 在任务列表中找到目标任务
3. 点击任务右侧的开关按钮
4. 开关变绿表示已启用，变灰表示已禁用

**说明**: 禁用的任务不会自动执行，但配置保留

#### 3.4 编辑定时任务
**完整路径**:
1. 点击左侧 ⏰ 定时任务图标
2. 点击要编辑的任务卡片
3. 修改任务信息（名称、Cron、提示词、技能）
4. 点击"保存"按钮

#### 3.5 手动执行定时任务
**完整路径**:
1. 点击左侧 ⏰ 定时任务图标
2. 找到要执行的任务
3. 点击"立即执行"按钮
4. 任务会立即运行一次，不影响定时计划

#### 3.6 删除定时任务
**完整路径**:
1. 点击左侧 ⏰ 定时任务图标
2. 找到要删除的任务
3. 点击删除按钮（垃圾桶图标）
4. 确认删除操作

#### 3.7 查看任务执行历史
**完整路径**:
1. 点击左侧 ⏰ 定时任务图标
2. 点击任务卡片
3. 在详情页查看"执行历史"标签
4. 显示最近的执行时间、状态、结果

## 第二部分：功能详细说明

### 5. 工具系统（内置功能）

#### 5.1 文件操作工具
- **read_file**: 读取文件内容
  - 用法: `read_file(path='文件路径')`
  - 支持指定行范围: `start_line=10, end_line=20`
  
- **write_file**: 写入文件
  - 用法: `write_file(path='文件路径', content='内容')`
  - 模式: 默认覆盖，可指定 `mode='append'` 追加
  
- **list_directory**: 列出目录内容
  - 用法: `list_directory(path='目录路径')`
  - 可递归: `recursive=True, max_depth=3`
  
- **search_files**: 搜索文件
  - 用法: `search_files(pattern='*.py', query='关键词')`
  - 支持正则表达式

#### 5.2 命令执行工具
- **execute_command**: 执行 Shell 命令
  - 用法: `execute_command(command='ls -la')`
  - 工作目录: 默认使用配置的工作空间
  - 超时: 默认 30 秒

#### 5.3 网络工具
- **web_search**: 网络搜索
  - 用法: `web_search(query='搜索关键词')`
  - 返回搜索结果摘要
  
- **send_email**: 发送邮件
  - 用法: `send_email(to='email@example.com', subject='主题', body='内容')`
  - 需要先配置 SMTP 设置

#### 5.4 实用工具
- **calculator**: 计算器
  - 用法: `calculator(expression='2 + 2 * 3')`
  
- **get_current_time**: 获取当前时间
  - 用法: `get_current_time(timezone='Asia/Shanghai')`
  
- **get_weather**: 查询天气
  - 用法: `get_weather(city='北京')`
  - 需要配置天气 API Key

#### 5.5 图像工具
- **create_image**: 生成图片
  - 用法: `create_image(prompt='图片描述', size='1024x1024')`
  - 需要配置图片生成 API
  
- **analyze_image**: 分析图片
  - 用法: `analyze_image(image_path='图片路径', question='问题')`
  - 需要配置图片分析 API

#### 5.6 地图工具
- **search_map**: 地图搜索
  - 用法: `search_map(query='附近的咖啡馆', location='北京')`
  - 需要配置地图 API Key

### 6. 技能系统（扩展功能）

**重要概念**: 技能不是工具！技能是包含命令行示例的文档，需要先读取文档，再执行命令。

#### 6.1 agent-browser（浏览器自动化）
**用途**: 自动化浏览器操作，抓取网页数据
**配置要求**: 
- `headless`: true/false（是否无头模式）
- `timeout`: 超时时间（秒）

**使用流程**:
1. 用户说"帮我抓取某网站的数据"
2. AI 读取: `read_file(path='skills/agent-browser/SKILL.md')`
3. 查看文档中的命令示例
4. 执行: `exec(command='python skills/agent-browser/scripts/browse.py --url=...')`

**常见问题**: 
- 浏览器启动失败 → 检查是否安装 Chrome/Chromium
- 抓取失败 → 尝试关闭 headless 模式

#### 6.2 baidu-search（百度搜索）
**用途**: 使用百度搜索引擎进行中文搜索
**配置要求**: 无需配置，开箱即用

**使用流程**:
1. 用户说"百度搜索某关键词"
2. AI 读取: `read_file(path='skills/baidu-search/SKILL.md')`
3. 执行: `exec(command='python skills/baidu-search/scripts/search.py "关键词"')`

#### 6.3 cron-manager（定时任务管理）
**用途**: 通过命令行管理定时任务
**配置要求**: 无需配置

**使用流程**:
1. 用户说"列出所有定时任务"
2. AI 读取: `read_file(path='skills/cron-manager/SKILL.md')`
3. 执行: `exec(command='python skills/cron-manager/scripts/list.py')`

**说明**: 通常用户会通过 UI 管理定时任务，此技能用于脚本化管理

#### 6.4 email（邮件发送）
**用途**: 发送邮件
**配置要求**:
- `smtp_host`: SMTP 服务器地址
- `smtp_port`: 端口（通常 587 或 465）
- `smtp_user`: 邮箱账号
- `smtp_password`: 邮箱密码或授权码
- `from_email`: 发件人邮箱

**使用流程**:
1. 用户说"发邮件给某人"
2. AI 读取: `read_file(path='skills/email/SKILL.md')`
3. 执行: `exec(command='python skills/email/scripts/send.py --to=... --subject=... --body=...')`

**常见问题**:
- 发送失败 → 检查 SMTP 配置是否正确
- 认证失败 → 使用授权码而非密码（如 QQ 邮箱）

#### 6.5 image-analysis（图片分析）
**用途**: 分析图片内容，提取信息
**配置要求**:
- `api_key`: 图片分析 API 密钥
- `api_provider`: 提供商（OpenAI/Claude/本地）

**使用流程**:
1. 用户说"分析这张图片"
2. AI 读取: `read_file(path='skills/image-analysis/SKILL.md')`
3. 执行: `exec(command='python skills/image-analysis/scripts/analyze.py --image=... --question=...')`

#### 6.6 image-gen（图片生成）
**用途**: 根据文字描述生成图片
**配置要求**:
- `api_key`: 图片生成 API 密钥
- `api_provider`: 提供商（DALL-E/Midjourney/Stable Diffusion）
- `default_size`: 默认尺寸（如 1024x1024）

**使用流程**:
1. 用户说"生成一张图片"
2. AI 读取: `read_file(path='skills/image-gen/SKILL.md')`
3. 执行: `exec(command='python skills/image-gen/scripts/generate.py --prompt="..." --size=1024x1024')`

**常见尺寸**: 1024x1024, 1024x1792, 1792x1024

#### 6.7 news（新闻获取）
**用途**: 获取最新新闻资讯
**配置要求**:
- `api_key`: 新闻 API 密钥
- `sources`: 新闻源列表
- `language`: 语言（zh/en）

**使用流程**:
1. 用户说"今天有什么新闻"
2. AI 读取: `read_file(path='skills/news/SKILL.md')`
3. 执行: `exec(command='python skills/news/scripts/fetch.py --category=tech --limit=10')`

#### 6.8 weather（天气查询）
**用途**: 查询天气信息
**配置要求**:
- `api_key`: 天气 API 密钥（如和风天气）
- `default_city`: 默认城市

**使用流程**:
1. 用户说"北京天气怎么样"
2. AI 读取: `read_file(path='skills/weather/SKILL.md')`
3. 执行: `exec(command='python skills/weather/scripts/query.py --city=北京')`


#### 6.9 小红书技能组（xhs-publisher / xhs-search / xhs-stats）

**重要**: 小红书技能需要先登录才能使用

##### 登录流程（所有小红书技能通用）
1. 打开终端，运行: `python skills/xhs-stats/scripts/login_helper.py`
2. 终端会显示二维码或登录链接
3. 使用小红书 App 扫码登录
4. 登录成功后会在 `skills/xhs-stats/scripts/core/` 目录生成 cookie 文件
5. 其他小红书技能会自动使用这个 cookie

**xhs-publisher（小红书发布）**
**用途**: 发布笔记到小红书
**配置要求**: 已登录（有 cookie 文件）

**使用流程**:
1. 用户说"发布笔记到小红书"
2. AI 读取: `read_file(path='skills/xhs-publisher/SKILL.md')`
3. 准备内容（标题、正文、图片）
4. 执行: `exec(command='python skills/xhs-publisher/scripts/publish.py --title="..." --content="..." --images="..."')`

**关键注意事项**:
- 换行符使用 `\n`（不是 `\\n`）
- 图片路径用逗号分隔: `image1.jpg,image2.jpg`
- 图片会自动上传到小红书
- 发布后返回笔记链接

**xhs-search（小红书搜索）**
**用途**: 搜索小红书笔记
**配置要求**: 已登录

**使用流程**:
1. 用户说"搜索小红书上的某内容"
2. AI 读取: `read_file(path='skills/xhs-search/SKILL.md')`
3. 执行: `exec(command='python skills/xhs-search/scripts/search.py --keyword="..." --limit=20')`

**xhs-stats（小红书数据统计）**
**用途**: 获取笔记数据、用户数据
**配置要求**: 已登录

**使用流程**:
1. 用户说"查看我的小红书数据"
2. AI 读取: `read_file(path='skills/xhs-stats/SKILL.md')`
3. 执行: `exec(command='python skills/xhs-stats/scripts/stats.py --type=notes')`

- 确认表达式格式正确: `秒 分 时 日 月 周`

**步骤 3: 检查调度器状态**
- 运行诊断命令: `python backend/scripts/check_cron_jobs.py`

- 后台调度器未启动（重启应用解决）
- 技能未启用或配置错误

#### Q6: Cron 表达式怎么写？
**格式**: `秒 分 时 日 月 周`

**常用示例**:
- `0 0 9 * * *`: 每天 9:00
- `0 30 8 * * 1-5`: 工作日 8:30
- `0 0 */2 * * *`: 每 2 小时
- `0 */30 * * * *`: 每 30 分钟
- `0 0 0 1 * *`: 每月 1 号 0:00
- `0 0 9 * * 1`: 每周一 9:00

**字段说明**:
- 秒: 0-59
- 分: 0-59
- 时: 0-23
- 日: 1-31
- 月: 1-12
- 周: 0-6（0=周日）

**特殊符号**:
- `*`: 任意值
- `*/n`: 每 n 个单位
- `n-m`: 范围
- `n,m`: 列表

### 12. 多智能体团队相关问题

#### Q11: 什么时候应该使用多智能体团队？
**适用场景**:
- 需要多角色协作的复杂任务（如内容创作：撰稿→审核→优化）
- 需要多视角分析的决策任务（如技术方案评估：性能视角、安全视角、成本视角）
- 需要专业分工的项目（如软件开发：需求分析→设计→编码→测试）

**不适用场景**:
- 简单问答（如"今天天气"、"帮我搜索"）
- 单一任务（如"写一段代码"、"翻译这段文字"）
- 快速响应场景（团队协作需要更多时间）

**判断原则**: 如果任务可以由单个 AI 快速完成，就不需要使用团队

#### Q12: 三种团队模式有什么区别？
**流水线模式（Pipeline）**:
- 成员按顺序依次处理
- 前一个成员的输出是下一个成员的输入
- 适合有明确流程的任务
- 示例: 撰稿人 → 审核员 → 优化师

**依赖图模式（Graph）**:
- 成员根据依赖关系工作
- 可以并行执行无依赖的任务
- 适合复杂的工作流
- 示例: 数据收集 → (分析 + 可视化) → 报告生成

**多视角模式（Council）**:
- 成员独立分析同一问题
- 最后汇总所有视角的结论
- 适合需要多角度评估的决策
- 示例: 技术可行性 + 成本分析 + 用户体验 → 综合决策

**交叉评审 vs 独立评审**（仅多视角模式）:
- 交叉评审: 成员可以看到其他成员的分析，互相参考
- 独立评审: 成员完全独立分析，避免互相影响

**步骤 4: 添加成员**

  - 角色名称: 如"技术撰稿人"

### 15. 工作空间和文件操作问题

#### Q20: 工作空间是什么？
**概念**: 工作空间是 AI 执行文件操作和技能命令的默认目录。

**作用**:
- 所有文件读写操作默认在工作空间进行
- 技能脚本在工作空间中执行
- 临时文件存储在 `{工作空间}/temp/`

**配置方式**:
1. 设置 → 工作空间 → 选择文件夹
2. 选择一个常用的项目目录
3. 保存后立即生效

**默认值**: 应用安装目录

#### Q21: 如何让 AI 操作特定目录的文件？
**方法一：配置工作空间**
- 将工作空间设置为目标目录
- 之后所有操作都在该目录进行

**方法二：使用绝对路径**
- 在命令中指定完整路径
- 如: `read_file(path='C:/Users/用户名/Documents/file.txt')`

**方法三：使用相对路径**
- 相对于工作空间的路径
- 如工作空间是 `C:/Projects/`，使用 `read_file(path='myapp/src/main.py')`

#### Q22: 临时文件存储在哪里？
**位置**: `{工作空间}/temp/`

**用途**:
- AI 生成的临时文件
- 下载的文件
- 中间处理结果

**清理**: 可以手动删除 temp 目录中的文件，不影响系统运行

### 18. 错误处理和故障排查

#### 18.1 应用启动失败
**可能原因**:
1. 端口被占用（默认 5000）
2. 数据库文件损坏
3. Python 依赖缺失

**排查步骤**:
1. 检查端口: `netstat -ano | findstr :5000`（Windows）
2. 检查数据库: 运行数据库诊断命令
3. 重装依赖: `pip install -r requirements.txt`
4. 查看日志: 打开 `botpy.log` 查看错误信息

#### 18.2 技能执行失败
**可能原因**:
1. 技能未启用
2. 配置参数错误
3. Python 依赖缺失
4. API 密钥无效

**排查步骤**:
1. 确认技能已启用（左侧技能面板，开关为绿色）
2. 检查配置（点击技能卡片查看 config.json）
3. 安装依赖: `pip install -r skills/{技能名}/requirements.txt`
4. 测试 API: 使用技能文档中的测试命令
5. 查看错误信息: AI 执行技能时会显示错误详情

#### 18.3 定时任务不执行
**完整排查流程**:

**检查点 1: 任务状态**
- 路径: 左侧 ⏰ → 找到任务
- 确认开关为绿色（启用）
- 确认 Cron 表达式正确

**检查点 2: 调度器状态**
- 运行: `python backend/scripts/check_cron_jobs.py`
- 确认输出显示"调度器运行中"
- 如果未运行，重启应用

**检查点 3: 执行历史**
- 点击任务卡片 → 执行历史
- 查看最近的执行记录
- 如果有错误，查看错误详情

**检查点 4: 技能依赖**
- 如果任务使用了技能，确认技能已启用
- 路径: 左侧 🔧 → 检查技能状态

**检查点 5: 系统日志**
- 查看 `botpy.log`
- 搜索任务名称或 "cron" 关键词
- 根据日志信息定位问题

**常见原因及解决**:
- Cron 表达式错误 → 使用 https://crontab.guru/ 验证
- 调度器未启动 → 重启应用
- 技能未启用 → 启用相关技能
- 提示词错误 → 修改任务提示词，确保清晰明确

#### 18.4 渠道消息收不到
**排查步骤**:

**步骤 1: 检查渠道配置**
- 设置 → 渠道 → 选择对应渠道
- 确认所有必填字段已填写
- 点击"测试连接"验证

**步骤 2: 检查机器人权限**
- 飞书: 确认机器人已添加到群组，且有消息权限
- QQ 频道: 确认机器人在频道中，且有发言权限
- 微信: 确认公众号已配置服务器 URL

**步骤 3: 检查网络连接**
- 确认服务器可以访问外网
- 检查防火墙设置
- 测试 API 端点是否可达

**步骤 4: 查看日志**
- 打开 `botpy.log`
- 搜索渠道名称（如 "feishu"、"qq"）
- 查看是否有错误信息

**步骤 5: 重启渠道服务**
- 设置 → 渠道 → 点击"重启服务"
- 或完全重启应用

### 19. 性能和优化问题

#### Q23: AI 响应很慢怎么办？
**可能原因及解决**:

**原因 1: 模型参数设置过高**
- 检查: 设置 → 模型配置 → Max Tokens
- 解决: 降低 Max Tokens 到 2000-4000
- 说明: Token 越多，生成时间越长

**原因 2: 使用了大模型**
- 检查: 设置 → LLM 提供商 → 查看当前模型
- 解决: 切换到更快的模型（如 GPT-3.5 而非 GPT-4）
- 说明: 大模型更准确但更慢

**原因 3: 网络延迟**
- 检查: 测试网络连接速度
- 解决: 使用国内 API 端点或代理
- 说明: 国外 API 可能较慢

**原因 4: 技能执行耗时**
- 检查: 查看 AI 正在执行的技能
- 解决: 某些技能（如浏览器自动化）本身就慢，属于正常
- 说明: 可以在技能配置中调整 timeout

**原因 5: 记忆搜索过多**
- 检查: 查看 `data/memory/` 目录大小
- 解决: 清理旧的记忆文件
- 说明: 记忆文件过多会影响搜索速度

#### Q24: 如何优化 AI 回答质量？
**优化方法**:

**方法 1: 调整温度参数**
- 路径: 设置 → 模型配置 → Temperature
- 精确任务: 0.0-0.3（如代码生成、数据分析）
- 平衡任务: 0.5-0.7（如写作、翻译）
- 创意任务: 0.8-1.2（如创意文案、头脑风暴）

**方法 2: 配置人设**
- 路径: 设置 → 人设
- 选择合适的预设性格
- 或自定义系统提示词，明确 AI 的角色和风格

**方法 3: 使用更好的模型**
- 路径: 设置 → LLM 提供商
- 选择更强大的模型（如 GPT-4、Claude 3）
- 说明: 更好的模型通常回答质量更高

**方法 4: 提供更清晰的提示**
- 明确任务目标
- 提供必要的上下文
- 分步骤描述复杂任务

**方法 5: 使用多智能体团队**
- 对于复杂任务，使用团队协作
- 多角色分工可以提高输出质量

### 23. 回答原则和风格

#### 23.1 回答结构
1. **先给操作路径**: 直接告诉用户怎么做
2. **再解释原理**: 简要说明为什么这样做
3. **提供补充信息**: 相关的注意事项或替代方案
4. **询问后续需求**: 是否需要进一步帮助

4. 建议实验: "我们可以尝试..."

**如果用户问题不清楚**:
1. 澄清问题: "你是想问如何配置渠道，还是如何使用渠道？"
2. 提供选项: "你可以选择：A) ... B) ..."

### 27. 故障排查决策树

#### 决策树 1: 技能不工作
```
技能不工作
├─ 技能是否启用？
│  ├─ 否 → 左侧🔧 → 找到技能 → 打开开关
│  └─ 是 → 继续
├─ 配置是否正确？
│  ├─ 否 → 点击技能卡片 → 编辑配置 → 保存
│  └─ 是 → 继续
├─ 依赖是否安装？
│  ├─ 否 → pip install -r skills/{技能名}/requirements.txt
│  └─ 是 → 继续
└─ 查看错误日志 → 根据错误信息处理
```

#### 决策树 2: 定时任务不执行
```
定时任务不执行
├─ 任务是否启用？
│  ├─ 否 → 左侧⏰ → 找到任务 → 打开开关
│  └─ 是 → 继续
├─ Cron 表达式是否正确？
│  ├─ 否 → 编辑任务 → 修改 Cron → 保存
│  └─ 是 → 继续
├─ 调度器是否运行？
│  ├─ 否 → 重启应用
│  └─ 是 → 继续
├─ 技能是否可用？
│  ├─ 否 → 启用相关技能
│  └─ 是 → 继续
└─ 查看执行历史 → 根据错误处理
```

#### 决策树 3: 渠道消息收不到
```
渠道消息收不到
├─ 渠道是否配置？
│  ├─ 否 → 设置 → 渠道 → 配置并保存
│  └─ 是 → 继续
├─ 测试连接是否成功？
│  ├─ 否 → 检查配置参数 → 重新测试
│  └─ 是 → 继续
├─ 机器人是否在群组中？
│  ├─ 否 → 添加机器人到群组
│  └─ 是 → 继续
├─ 机器人是否有权限？
│  ├─ 否 → 在平台设置中授予权限
│  └─ 是 → 继续
└─ 查看日志 → 根据错误处理
```

### 28. 进阶技巧

#### 28.1 组合使用多个功能
**示例 1: 定时任务 + 技能 + 渠道**
- 创建定时任务，使用新闻技能获取资讯
- AI 自动总结新闻
- 通过渠道发送到飞书群

**示例 2: 团队 + 技能**
- 创建内容创作团队
- 团队使用图片生成技能创建配图
- 团队使用小红书技能发布内容

**示例 3: 记忆 + 个性化**
- AI 记住用户的偏好和习惯
- 根据记忆提供个性化建议
- 自动调整回答风格和内容

#### 28.2 优化工作流程
**技巧 1: 使用工作空间**
- 将常用项目设为工作空间
- 所有文件操作自动在该目录进行
- 避免每次都指定完整路径

**技巧 2: 创建专用团队**
- 为常见任务创建专用团队
- 如"代码审查团队"、"文档团队"
- 提高任务执行效率和质量

**技巧 3: 合理使用定时任务**
- 周期性任务用定时任务自动化
- 如每日报告、定期备份、数据同步
- 减少重复手动操作

**技巧 4: 善用记忆功能**
- 让 AI 记住项目信息、配置偏好
- 后续对话无需重复说明
- 提高沟通效率
